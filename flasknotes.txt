1. how does flask work?

In the PasteLink project, the HTML file is the frontend that the user sees and interacts with, while Flask acts as the backend where all the logic is written. When Flask starts, it runs a server on `http://127.0.0.1:5000`, where `127.0.0.1` means "my own computer" (localhost) and `5000` is the port Flask is listening on. Every request first reaches this server, and Flask then checks the route (the part after the domain). For example, `/` is usually the homepage route, so visiting `http://127.0.0.1:5000/` runs the function attached to `@app.route("/")`, which returns the homepage HTML. When the user clicks the Generate button, the form sends a request to `/generate`, and Flask runs the function attached to `@app.route("/generate")`. That function contains the backend logic, such as reading the pasted text, generating an ID, storing the data, and returning a response. The entire project runs on the same server, and different features are accessed through different routes like `/`, `/generate`, and `/paste/<id>`. In simple terms, the server is the central hub, routes are like doors, and Flask decides which function to run based on which door (URL) the user enters.
